Subneteo de red con VLSM


Resumen

El subneteo o subnetting se refiere a la división de una red IP física en múltiples subredes lógicas con el objetivo de que cada una de estas trabajen a nivel de envio y recepcion de paquetes con una red independiente, aunque todas se encuentren en la misma red física y el mismo dominio, el subneteo permite una mejor administración, control de tráfico y seguridad al segmentar una red por funciones específica, además de que se recomienda el subneteo de una red porque de esta manera se mejora el rendimiento de esta al minimizar el tráfico a nivel broadcast dentro del dominio de red, el subneteo funciona tomando y convirtiendo los bits de la porción de Host para así administrarlos con el fin de realizar los segmentos correspondientes dentro de una misma red.

Antes de continuar, debemos verificar la topología de red, para analizarla y así poder seguir con el procedimiento:


En primera medida debemos analizar los dispositivos presentes en la topología, donde podremos encontrar:

  • Servidores (No tenidos en cuenta para el desarrollo del ejercicio)
  • Routers: Estos son dispositivos de capa 3 los cuales se encargan del enrutamiento de los datos (MODELO OSI Y DISPOSITIVOS de RED POR CAPA, n.d.)
  • Switches:Estos son dispositivos de capa 2 los cuales se encargan de la transmisión de los datos dentro de una red de dispositivos (MODELO OSI Y DISPOSITIVOS de RED POR CAPA, n.d.)
  • Multilayer Switch: los switches multicapa se implementan en las capas de núcleo y de distribución de la red conmutada de una organización. Los switches multicapa se caracterizan por la capacidad de crear una tabla de routing, por admitir algunos protocolos de routing y por reenviar los paquetes IP a una velocidad similar a la de reenvío de capa 2. Los switches multicapa suelen admitir hardware especializado, como los circuitos integrados de aplicación específica (1.2.1.5 Switching Multicapa, 2022), además de esto, es importante destacar que estos dispositivos ahorran a nivel monetario, porque poseen la capacidad de transmitir los datos entre la capa 2 y 3, por eso afirmo y cito: "Los switches multilayer pueden operar tanto en Capa 2 como en Capa 3 (InterVLAN). En Capa 2 funciona entre interfaces asignados a VLANs de Capa 2 o a Trunks de Capa 2. En Capa 3 funciona entre cualquier tipo de interface mientras esta tenga una IP asignada. Un switch multilayer puede asigna una IP a una interface física lo mismo que un router aunque también puede asigna una IP a una interface lógica (que se llaman Switched Virtual Interface SVI) que representa la VLAN entera. La IP que se configura se convierte en el Gateway por defecto para cualquier servidor que esté conectado a la interface o VLAN, con lo cual el servidor usará la interface de Capa 3 para comunicarse con el exterior de su dominio de broadcast" (Iñaki Martínez Díez I.M.D, 2018).
  • PC: Conjunto de circuitos electrónicos encargados de generar peticiones y recibir respuestas dentro de una topología de red.
